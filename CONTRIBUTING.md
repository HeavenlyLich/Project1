# 🛠 Порядок роботи з проєктом (Contributing Guide)

Дякуємо, що виявили інтерес до покращення нашої гри! Щоб зробити процес розробки структурованим, ми використовуємо наступний алгоритм:

### 1. Пошук завдань
* Ознайомтеся зі списком відкритих **Issues**.
* Якщо ви знайшли новий баг — створіть новий Bug Report.

### 2. Робота з гілками (GitFlow)
* Створюйте окрему гілку для кожного виправлення:
  `git checkout -b fix-your-bug-name`
* Назви комітів мають бути лаконічними та англійською мовою.

### 3. Написання тестів
* Кожна нова функція або виправлення багу має супроводжуватися відповідним тестом у `test_pytest_modern.py`.
* Перед відправкою коду переконайтеся, що всі існуючі тести проходять успішно.

### 4. Pull Requests
* Після завершення роботи створіть Pull Request у гілку `main`.
* Детально опишіть, що саме було змінено та який Issue це закриває (наприклад, `Closes #4`).

---
*Разом ми зробимо цей код кращим!* 🚀
